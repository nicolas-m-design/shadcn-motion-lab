# Writing Motion Specs

Use compact specs that are explicit enough for implementation and review.

## Suggested fields

- Component
- Variant
- Intent
- Trigger
- Behavior
- Timing
- Easing
- Spatial movement
- Staging
- Accessibility
- Reduced motion
- Implementation notes
- Reliability check

## Standard

- Describe what the motion should communicate, not just the CSS properties.
- State whether entry and exit differ.
- State whether interruption matters.
- State the reduced-motion behavior directly.
- State when not to use a motion library.
